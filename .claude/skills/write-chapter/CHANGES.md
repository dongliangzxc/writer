# Context Agent 细纲约束改进 - 修改记录

## 问题
Context Agent 无法读取细纲表格格式（`大纲/卷一细纲.md`），导致：
- 无法获取【硬】【中】【软】约束级别
- Step 2A 起草时缺乏硬性约束，容易偏离主线

## 修改内容

### 1. context-agent.md（上下文搜集Agent）

#### 新增 Step 0.4: 细纲解析（关键步骤）
- **位置**：Step 0 之后，Step 0.5 之前
- **功能**：
  - 优先读取 `大纲/卷{N}细纲.md` 格式
  - 从表格提取当前章的：
    - 必须发生的事件
    - 级别（【硬】/【中】/【软】）
    - 状态/说明（偏差预警）
  - 将【硬】约束标记为"不可变事实"

#### 约束级别定义
- **【硬】约束**：必须严格遵循，不可删除/替换
- **【中】约束**：主线方向不变，实现方式可调整
- **【软】约束**：可按写作需要灵活处理

#### 更新 Step 1: 读取大纲策略
- 添加细纲表格格式作为第一优先级
- 添加降级读取策略（单章文件 → 卷级大纲 → 总纲）

#### 更新 Step 5: 组装创作执行包
- 任务书第1板块包含细纲约束：
  - 【硬】约束列表
  - 【中】约束列表
  - 【软】约束列表
  - 偏差预警

#### 更新 Step 6: 逻辑红线校验
- 新增红线7：细纲约束偏离检查
- 检查是否处理【硬】约束或擅自更改主线事件

### 2. SKILL.md（主流程）

#### 更新 Step 2A: 正文起草
- 添加遵循细纲【硬】约束的要求
- 添加尊重细纲【中】约束的要求
- 添加参考细纲【软】约束的要求

### 3. structural-checker.md（结构性审查）

#### 新增第四层检查：细纲约束一致性
- **【硬】约束偏离**（critical）：
  - 必须发生事件未出现
  - 主线事件被删除/替换
- **【中】约束偏离**（high）：
  - 主线方向改变
  - 关键角色未按细纲出场

#### 更新输出 metrics
- 添加 `outline_deviations` 字段

## 预期效果

1. **第1阶段（Context Agent）**：
   - 能正确读取细纲表格
   - 提取【硬】约束作为"不可变事实清单"

2. **第2阶段（起草）**：
   - Step 2A 明确知道哪些事件不能改
   - 在约束框架内发挥创意

3. **第3阶段（审查）**：
   - structural-checker 检查是否偏离细纲
   - critical 级别问题阻止偏离

4. **结果**：
   - 【硬】约束事件必须完成，不偏离主线
   - 【中】约束主线保留，细节可发挥
   - 【软】约束可灵活调整

## P0 修复记录

### P0-1: data-agent 保存细纲约束

**新增 Step D.5: 细纲约束执行记录**
- 在正文中搜索【硬】约束关键词
- 判断约束完成情况：completed / partial / missed / deviated
- 生成 `outline_constraints` 记录写入 state.json
- 偏离时写入 `deviation_warning`

**更新 chapter_meta 接口规范**
- 添加 `outline_constraints` 字段：
  - `hard_completed`: 已完成的硬约束列表
  - `hard_missed`: 未完成的硬约束列表
  - `deviation_warning`: 偏离警告

### P0-2: polish-guide 保护【硬】约束

**更新执行顺序**
- 第1步：【硬】约束检查（最先执行）- 标记约束位置
- 第7步：【硬】约束复核（最后执行）- 确保润色后约束仍在

**更新润色红线**
- 新增最高优先级红线："不删除【硬】约束事件"

**更新 checklist**
- 新增检查项：【硬】约束保护检查通过

### 约束保护完整链路

```
Context Agent (提取约束)
    ↓
Step 2A (起草遵循约束)
    ↓
Step 3 审查 (检查偏离)
    ↓
Step 4 润色 (保护约束不被删)
    ↓
Data Agent (记录约束完成)
    ↓
下章 Context Agent (读取上章约束记录)
```
