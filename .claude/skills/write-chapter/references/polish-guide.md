## 2. 执行顺序（必须按序）

> **润色轮次：1 轮完成。**（2026-04-25 从 2 轮改为 1 轮：实测第 2 轮改动极少但要付出完整重读代价，性价比过低。）1 轮结束后无论是否全部修复，**必须停止并进入 Step 5**，未修复项记入变更摘要的"遗留问题"节。

1. **保护【硬】约束检查（最先执行）**
   - 从审查报告或细纲读取本章【硬】约束事件，标记位置
   - 润色时绝对禁止删除或替换【硬】约束事件

2. **引用 Step 3 Part B 的扫描报告**（chapter_scan.py 已在 Step 3 执行完毕，此处直接消费其输出）

   **扫描报告处理规则**：
   - **ABC 层（强制修改）**：必须在润色中修复
   - **D 层（建议修改）**：尽量修复，修不完的记入"遗留问题"
   - A 层（短语）：角色名/专有名词可保留，其余每项最多保留 2 次
   - B 层（句式词）：「像是」「某种」等每项全章最多 3 次，其余必须改为具体描写或直接删除
   - C 层（句式模式）：同一句式结构最多 2 次，其余必须用完全不同的句式替换
   - D 层（内心独白注水，**建议层**）：
     - 收束词（不知道/只知道/没有说话/她想/他想）每项全章最多 2 次
     - "还是……？"排比追问最多保留 1 组（≤3个分支），其余删除或改为具体推理
     - 跨段复述同一短语：已知信息只在首次出现时完整写，后续一笔带过
     - 连续纯思绪段（≥3段无动作对话）：每 2 段后必须插入动作/感官/对话打断
     - 超长段落（>150字）：建议拆分为多个短段落
   - 替换方向：用具体的、不同的动作/感官描写替代，禁止只换近义词
   - 若某个高频表达是角色口头禅或有意为之的修辞，记入 `deviation`

3. 修复审查报告中的问题（先 `critical/high`）
4. **风格转译**（§3）
5. **Anti-AI + No-Cliché 终检**（§4）
6. **No-Poison 毒点规避**（§5）
7. **【硬】约束复核（最后执行）**
   - 润色后检查：所有【硬】约束事件是否仍然完整存在
   - 若发现被误删/替换：恢复原文并记录 deviation
8. 输出润色结果与 deviation（若有）

## 3. 风格转译

**禁改项（红线）**：剧情走向、事件顺序、角色行为结果、设定/能力描述、伏笔内容。

**硬约束**：
- 长句（>40字）拆分，避免连续长句压读
- 抽象判断 → 动作/反应/代价
- 删除"总结式旁白"和大段纯解释（连续>200字）
- 章内至少有 1 个明确推进点（信息/行动/关系/局势至少一项）

**软建议**：
- 开头尽早进入冲突/风险/强情绪（建议前 200-400 字）
- 后段或章末设置未闭合问题/期待锚点
- 微兑现按章型安排 1-3 次，避免机械等间距
- 对话保持"意图冲突"，减少无效寒暄

**分题材加权**：
- 言情/替身/狗血：情绪弧线前置，关键场景必须有关系位移
- 玄幻/修仙/高武：动作与结果比重更高，术语解释后置
- 都市/直播/电竞：信息节奏更快，强化"反馈-反应-反制"三连
- 悬疑/规则怪谈：线索投放要可回收，恐惧来自规则而非堆词

**过渡章适配**（由章纲/卷纲决定，禁止用字数判定）：冲突导入可放宽到前400字，微兑现降为0-1次，期待锚点强度可为weak。

## 4. Anti-AI + No-Cliché 终检

> Step 4 完成后必须输出 `anti_ai_force_check: pass/fail`。fail 时记入"遗留问题"继续进入 Step 5（受1轮润色限制）。

### 4.1 Anti-AI 高风险词库（命中即改写或记录 deviation）

**总结归纳词**：综合、总之、总而言之、由此可见、归根结底、说到底、总体来看、换句话说、简而言之、可以说、总的来说、综上。

**枚举模板词**：首先、其次、再次、最后、第一、第二、第三、一方面、另一方面、此外、另外。

**书面学术腔**：某种程度上、本质上、层面上、体现为、构成了、形成了、实现了、进行了、展开了、呈现出、表现出、反映出、折射出。

**逻辑连接滥用**：因此、因而、然而、与此同时、同样地、进一步、从而、进而、于是、故而、相较之下。

**情绪直述词**：非常愤怒/开心/难过、五味杂陈、百感交集、情绪复杂、内心震撼、感到无奈/痛苦/欣慰/恐惧、心潮起伏、心情沉重/复杂。

**动作套话**：皱起眉头、叹了口气、深吸一口气、缓缓开口、沉声/淡淡/冷冷/轻声说道、嘴角上扬/抽了抽、身形一滞、脚步一顿、浑身一震、心头一跳、不由后退半步、猛地转身、缓缓点头、轻轻摇头。

**环境套话**：空气仿佛凝固、气氛骤然紧张、夜色如墨、月色如水、四周/死一般的寂静、时间仿佛静止、空间仿佛扭曲、压抑得让人喘不过气。

**叙事填充词**：事实上、实际上、某种意义上、严格来说、通常情况下、这意味着、这说明、不可否认、毋庸置疑。

**抽象空泛词**（单独使用无叙事功能时改写）：命运、成长、蜕变、升华、抉择、信念、初心。

**机械开场/收尾**：命运的齿轮开始转动、故事才刚刚开始、真正的考验还在后面、一场风暴即将来临、更大的阴谋正在酝酿、答案尚未揭晓、可他/她不知道的是、然而一切才刚开始。

### 4.2 Anti-AI 句式规则

- 禁止"首先/其次/最后"三段式说明写法
- 禁止连续三句"主语+谓语+宾语"同构句
- 禁止清单化叙事（"他需要做三件事：一、二、三"）
- 连续两个以上形容词修饰同一名词 → 重写
- 每300字"很/非常/极其/格外/十分"总数≤4
- 禁止"说明书式对话"（完整解释背景、逻辑过满），对话必须有意图
- 单句成段占比25-45%，段落长度20-100字
- 禁止连续 `......` 或 `！！！`，省略号/感叹号每段最多1次
- 长句中逗号超过4个 → 拆句
- 每500字四字套语≤3个，且不得连续出现

### 4.3 Anti-AI 改写算法

1. 抽象情绪句 → `生理反应 + 当下意图 + 下一动作`
2. 结论句 → `事实细节 + 代价/风险 + 决策`
3. 连续说明句（≥3）→ 改成"对白/动作/反问"混排
4. 连续同构句（≥3）→ 至少打断1句为短句或插入动作锚点
5. 对话整段解释背景 → 改为"带意图的对抗式对白"

### 4.4 No-Cliché 古言陈词黑名单

> Anti-AI 管"像机器写的"，No-Cliché 管"像模板古言写的"。

**单章限用表达（每项最多出现 1 次）**：

| 类别 | 黑名单表达 |
|------|-----------|
| 心脏反应 | 心头一紧/一跳/一颤/一凛/一震、心猛地一跳、心跳漏了一拍 |
| 呼吸反应 | 呼吸一滞、屏住了呼吸、呼吸微微急促 |
| 身体僵硬 | 浑身一僵、脚步一顿、身形一滞、身子微微一僵 |
| 攥紧动作 | 攥紧了帕子/袖口、手指收紧、握紧了拳头 |
| 目光深邃 | 目光深不见底、眼神幽深如潭、目光幽暗 |
| 笑意不达 | 笑意未达眼底、嘴角的弧度没有温度 |
| 瞳孔反应 | 瞳孔猛地收缩/微缩、眼瞳一震 |

**单章禁用句式（出现即改）**：
- "她不知道……"作为段末/章末独立句 → 用具体行动或感知替代
- "窗外，天色渐暗/月色如水/夜色沉沉"作为章末 → 以角色动作或对话收束
- "命运的齿轮开始转动""一切才刚刚开始" → 直接删除
- "她知道……但她也知道……"句式 → 拆成具体思考或行动

**跨章频率控制**（每3章内最多2次）：
- "目光在她脸上停留了一瞬" / "投下的阴影将她笼罩"
- 男主单字命令句（"坐。""说。"）—— 3章内最多1次
- "替她拢了拢耳边的碎发" —— 连续2章禁止重复

### 4.5 跨章重复模式检查

**禁止模式**（对照 `state.json → recent_openings/recent_endings` 最近3章）：

| 检查项 | 禁止 | 通过条件 |
|--------|------|---------|
| 时间线 | 连续3章同一时段开头/结尾 | 最近3章有时间推进 |
| 开头 | 连续3章"醒来/走进"同一模式 | 最近3章开头类型不同 |
| 场景 | 连续4章同一室内地点 | 每4章至少1章换场景 |
| 情绪收尾 | 连续3章"睡不着/窗外X影"收尾 | 最近3章收尾模式不同 |
| 梦境 | 连续2章类似梦境/闪回 | 间隔至少1章 |

**收尾禁止→替代**：
- "窗外暮色…沉沉睡去" → 事件预告收尾
- "躺在床上…睡不着" → 行动收尾（"她站起身…"）
- "窗外树影…轻轻晃动" → 事件打断收尾
- "她知道…却…" → 选择/决定收尾

## 5. No-Poison 毒点规避（5类）

1. **降智推进**：角色忽略常识仅为推进剧情
2. **强行误会**：可一句话说清却长期拖延
3. **圣母无代价**：无边界原谅高风险对象
4. **工具人配角**：只在功能节点出现，没有独立动机
5. **双标裁决**：同类行为评价标准不一致

命中任一毒点时，必须补"动机/阻力/代价"中的至少两项。

## 6. 审查问题修复优先级

| 严重度 | 处理规则 |
|---|---|
| critical | 必须修复；无法修复必须记录 deviation |
| high | 必须优先处理；无法修复记录 deviation |
| medium | 视篇幅和收益处理 |
| low | 可择优处理 |

类型修复：OOC→恢复角色口吻；POWER_CONFLICT→能力回落或补获得路径+代价；TIMELINE_ISSUE→补时间锚点；LOCATION_ERROR→补移动过程；PACING_IMBALANCE→增推进事件或删冗余；CONTINUITY_BREAK→补衔接。

## 7. 润色红线（不可突破）

- 不删除【硬】约束事件（最高优先级）
- 不改剧情走向（大纲即法律）
- 不改设定物理边界（设定即物理）
- 不删除关键伏笔
- 不强行改写角色关系基线
