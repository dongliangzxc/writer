# 上下文搜集与执行包组装

> Step 1 加载此文档后，主 Agent 按以下指引内联执行，不调用子 Agent。

## CLI 命令（按顺序执行）

```bash
# 上下文快照（优先复用已有快照）
python "${SCRIPTS_DIR}/webnovel.py" --project-root "${PROJECT_ROOT}" context -- --chapter {NNNN}

# Context Contract 上下文包
python "${SCRIPTS_DIR}/webnovel.py" --project-root "${PROJECT_ROOT}" extract-context --chapter {NNNN} --format json
```

**按需查询**（各命令只执行一次，不重试）：
```bash
# 最近章节模式（用于跨章重复检查）
python "${SCRIPTS_DIR}/webnovel.py" --project-root "${PROJECT_ROOT}" index get-recent-chapter-meta --limit 5

# 实体查询（按需）
python "${SCRIPTS_DIR}/webnovel.py" --project-root "${PROJECT_ROOT}" index get-core-entities
python "${SCRIPTS_DIR}/webnovel.py" --project-root "${PROJECT_ROOT}" index recent-appearances --limit 20
```

## 必须加载的文件

- 章节细纲：`大纲/卷N/第XXX章.md` 或 `大纲/第{卷}卷-详细大纲.md`
  - **必须读取本章所在弧的"弧时间节奏表"**：从细纲中找到当前章所在弧的时间表，提取本章对应的故事时段和时间模式，写入板块1的【时间模式】和【时间锚点】
- `state.json`：progress / protagonist_state / chapter_meta / project.genre / plot_threads.foreshadowing
- 上章（ch{NNNN-1}）完整正文
- 上上章（ch{NNNN-2}）最后 300 字
- 上章摘要：`.webnovel/summaries/ch{NNNN-1}.md`（缺失则降级为正文前 300-500 字）
- `CLAUDE.md`（提取感情线阶段锁、伏笔时机锁、道具注册表）
- 独立时间线表：`大纲/第{volume_id}卷-时间线.md`（旧格式，弧时间表优先；两者都无则从上章推断）

**Context Contract 消费**：必读 `writing_guidance.guidance_items`；推荐读 `reader_signal`；`rag_assist` 仅当 `invoked=true` 且 `hits` 非空时提炼为约束。

**缺失处理**：第1章无上章→跳过板块2；chapter_meta 不完整→只用现有数据；foreshadowing 缺失→板块4标注"伏笔数据缺失"。

## 执行包格式（5 板块，禁止扩展）

**1. 本章写什么（核心任务）**
- **核心三问**（首行）：① 核心问题（逼谁做什么选择）② 信息增量（读者新知道什么）③ 悬念增量（读者最想知道什么）。三问答不出说明本章无实质内容，须与用户确认。
- 一句话概括
- **【章形】**：{A/B/C/D/E/F}（从细纲读取）
- **【时间模式】**：{连续/跳跃/新开}（从时间线表或上章推断）
- **【场景延续状态】**：{延续上章/新开场景}
- **【场景类型】**：{日常场景/大场面切片}
- **【时间锚点】**：{具体时辰}
- 【硬】约束事件（从细纲提取，必须完成），用 ①②③ 编号
- 【中】约束事件（方向保持，细节可改），接续硬约束编号，每条指定信息载体
- 核心冲突 / 局面变化
- **章末最后画面**（从细纲章末钩转化）：画面内容 + 禁止在画面后追加感悟/总结/环境/夜眠，≤50字收章

**2. 从哪里接上（承接上章）**
- 上章结束时的场景快照：时间/地点/在场人物/情绪/未完结事件（从正文自然提取）
- 本章开头如何承接（一句话）
- **承接断裂禁止**（仅列本章适用项）：上章危机/行动截断→禁止跳到次日；上章对话未完→禁止跳过；连续2章入睡收尾→本章换结尾方式

**3. 出场角色**
- 每个角色一行：名字 / 当前状态 / 本章动机 / 情绪底色 / 红线
- 只列本章实际出场角色

**4. 不可变事实（红线）**
- 大纲事实 / 设定事实 / 承接事实
- 到期伏笔（remaining ≤ 5 或已超期）
- 感情线阶段约束（从 CLAUDE.md 提取）
- 道具约束（仅本章涉及的已注册道具）

**5. 禁止事项（不超过 9 条）**
- 跨章重复模式（从已加载的 chapter_meta 提取最近 3 章模式）
- 细纲红线 + 感情线阶段锁 + 伏笔时机锁
- 章形对应禁止约束
- 日弧孤岛禁止（连续3章"晨起→夜眠"）
- 场景延续/时间模式的强制约束

**禁止输出**：追读力策略、钩子建议、微兑现建议、场景质感清单、终检清单。

## 章形定义（A-F）

| 章形 | 名称 | 核心约束 |
|------|------|---------|
| A | 刺入式 | 中途切入事件，禁止晨起/梳妆开场 |
| B | 镜头切分式 | ≥2个场景切分，禁止单场景流水账 |
| C | 单线贯穿 | 时间线连续，禁止跳跃/压缩过渡 |
| D | 对话主导 | 对话占比>60%，禁止大段环境/心理描写 |
| E | 双线缠绕 | 两条情节线必须交叉≥1次 |
| F | 单点透视 | 聚焦单一物件/场景，禁止发散支线 |

## 时间模式定义

| 模式 | 约束 |
|------|------|
| 连续 | 继承上章时间锚点，禁止跳到次日 |
| 跳跃 | 一句话带过中间时段，必须用A型章形 |
| 新开 | 无需承接，但禁止连续3章晨起开场 |

## 红线校验（组装后、输出前）

任一 fail 则重组执行包：
1. 不可变事实冲突（大纲/设定/上章结果矛盾）
2. 能力或信息无因果来源
3. 角色动机断裂
4. 时间模式违规（仅时间线表存在时检查）
5. 大场面切片规划了闭环结尾
