# 卷级细纲写作（弧式结构·增量模式）

## 核心架构：弧级增量写入 + 断点续传

> **设计原因**：一次性生成 50 章细纲容易因上下文过长而中断，中断后全部丢失。因此改为**每完成一条弧就立即写入文件**，即使中途断开，已完成的弧也安全落盘。恢复时自动检测已写入的弧，从断点继续。

**写入时机**（硬规则）：
- Step 1 完成 → 立即写入文件头（标题 + 格式说明 + 弧结构总览表）
- 每条弧的 Step 2 完成 → 立即追加该弧的章节表格到文件
- 全部弧完成 → 追加三线索追踪表 + 设计摘要

**断点续传**（硬规则）：
- Step 0 检测到 `大纲/卷{N}细纲.md` 已存在时，解析已写入的弧编号
- 跳过已完成的弧，从第一条未写入的弧继续
- 弧结构总览表已存在则复用，不重新设计

## 目标

- 产出一份**弧式细纲**：每卷 5-10 条弧，每条弧横跨 2-5 章，每章在弧内占一个**半完成节拍**。
- 三线（感情线/悬疑线/宅斗线或当卷对应线）**在章内交叉**，不走平行独立轨道。
- 每章必须有明确的章末**悬置点**，感情线与事件线禁止同时闭合。
- 消除"日常过渡占位章"——所有章节都必须有实质推进，感情线日常融入事件章而不单独占章。
- 产物文件：`大纲/卷{N}细纲.md`，格式与卷一一致。

## 核心设计原则（必须执行）

### 原则一：弧式分段，禁止一事一章

每个"事件"必须跨至少 2 章。常见节拍分法：

| 弧节拍 | 典型章末状态 |
|--------|------------|
| 起（信息建立/威胁出现） | 悬置——新异常或新威胁未解 |
| 承（矛盾升温/行动受阻） | 悬置——行动被中断或代价升高 |
| 转（局面翻转/新变量介入） | 悬置——新问题比旧问题更大 |
| 合（局面收束，但留新缺口） | 悬置——当前弧结束，但下一条线已启动 |

> 注意："合"节拍允许局部收束，但必须同时开启下一弧的第一个异常，不允许全线清零。

### 原则零（最高优先级）：事件密度保障——每章必须有足够的行动素材

> **根因**：写作模型在大纲事件不足时，会用内心独白、多POV复述、情绪排比来凑字数，产生严重注水。因此细纲必须在设计阶段就保证每章有足够的"可写行动"。

**硬规则**：
- 每章【硬】列必须包含 **≥2 个独立行动节拍**（不同主语或不同动作链）
- 每个行动节拍必须是"主语+具体动作+可观察结果"格式，可以被拍成镜头
- 若某章的核心是一句关键台词（如"不用解释"），则必须在【硬】列补充该台词发生前后的行动链（谁做了什么导致这句话、说完后谁又做了什么）

**判断方法**：把【硬】列的所有节拍列出来，想象写作模型只写这些行动和对话，不加任何内心独白——能否撑到 1200 字以上？如果不能，说明事件不够，需要补充。

**反面教材（来自卷一实际问题）**：
```
✗ ch43【硬】：谢沉渊回府，谢明姝告状，他说"我见过"
→ 问题：苏绾音整章没有行动，只能写内心独白注水
→ 实际产出：3段重复"他到底知不知道"+4段排比追问+2段窗外景物

✓ ch43 修复后：①谢明姝向谢沉渊告状，他只说"我见过"便打发走 ②苏绾音从春桃处听说此事，起身走到窗边观察书房灯光 ③谢沉渊来西厢，说"不用解释"，说"下次有事告诉我" ④谢沉渊回书房，取出母妃遗物双鱼玉佩对比
→ 结果：4个行动节拍，1859字实质内容，零注水
```

### 原则二：三线交叉

每章必须至少有两条线同时在场（主线推进 + 另一条线有实质信号）：

- **感情线**：不单独占章。感情推进通过"两人在事件中的共同经历/对话/反应"完成，以旁白+行为暗示为主，不写独立"甜宠日常章"。
- **悬疑线**：每条伏笔有独立的铺垫→强化→误读→揭晓节奏，禁止整章只有悬疑、感情线完全消失。
- **宅斗/对抗线**：反派的行动要与感情线或悬疑线产生交叉，禁止反派章与主角感情章完全隔离。

### 原则三：读者视角差（必须规划）

每卷至少安排 2-3 处"读者知道、主角不知道"的信息差设计：
- 典型形式：章末最后一段切换视角（反派/谢沉渊侧），给读者一个主角不知道的信号
- 这类设计必须在细纲中明确标注 **【读者视角差】**

### 原则四：悬置点分类

章末悬置点必须属于以下类型之一（在表格"章末钩"列注明类型）：

| 类型标签 | 说明 |
|---------|------|
| 疑问未答 | 明确提出疑问，章内无答案 |
| 行动截断 | 某件事被打断或尚未执行 |
| 新异常 | 出现与预期不符的细节或信号 |
| 威胁升级 | 压力已升但未释放 |
| 信息错位 | 主角误读某件事，读者已知真相 |
| 视角切换 | 末段切至他人视角，给出读者信息差 |

### Step 1：弧结构设计 → 立即写入文件头

基于"卷级输入摘要"，设计本卷的弧结构。

**输出格式**：
```
弧编号 | 弧名 | 跨章范围 | 核心驱动事件 | 弧末状态（留给下一弧的缺口）
弧A    | [名] | ch51-54  | [事件]        | [悬置描述]
弧B    | [名] | ch55-58  | [事件]        | [悬置描述]
...
```

**设计规则**：
- 每卷 5-10 条弧，不强制对称
- 感情线阶段跃迁必须附着在某条事件弧上，不单独成弧
- 卷末最后一条弧必须以"悬置大钩"收束（为下卷开局埋线），禁止全卷闭合
- 大反派弧必须在卷中段已有铺垫，不允许最后一章突然登场

**立即写入文件头**（Step 1 完成后必须执行）：

用 Write 工具创建 `大纲/卷{N}细纲.md`，写入以下内容：

```markdown
# 卷{N}细纲（表格版）第{start}-{end}章

## 格式
- 【硬】=必须完成，不可删除
- 【中】=主线保持，细节可改
- 【软】=建议/章末钩（可调整）

---

## 弧结构总览

| 弧 | 弧名 | 章节范围 | 核心驱动 | 弧末缺口 |
|----|------|---------|---------|---------|
| 弧A | ... | ch51-54 | ... | ... |
| 弧B | ... | ch55-58 | ... | ... |
...

---

<!-- 以下章节细纲由逐弧增量写入，断点续传标记勿删 -->
```

**写入确认**：输出 `✅ 弧结构已写入 大纲/卷{N}细纲.md（共 X 条弧，接下来逐弧展开）`

### Step 2：逐弧循环（核心增量步骤）

**对每条弧（未完成的），依次执行以下子步骤**：

```
for 弧X in [未完成弧列表]:
    Step 2a: 章节节拍分配（仅本弧的章节）
    Step 2b: 弧内校验（三线交叉 + 钩子 + 孤岛 + 注水检查）
    Step 2c: 写入文件（追加本弧的章节表格）
    输出进度：「✅ 弧X 已写入（Y/Z 条弧完成）」
```

#### Step 2a：章节节拍分配（仅本弧）

将本弧拆分为章级节拍，逐章填入三列表格：

| 章 | 【硬】必须完成 | 【中】主线保持 | 【软】建议/章末钩子 |
|---|---------------|---------------|-------------------|

**填写规则**：

**【硬】列**：
- 本章在弧内的节拍位置（起/承/转/合）
- **≥2 个独立行动节拍**，用 ①② 编号，每个节拍格式为"主语+具体动作+可观察结果"
- 若有里程碑达成，标注 `**【chXX_里程碑名】里程碑**`
- **禁止感受型节拍**：【硬】列中不得出现以下写法——
  - "XX不确定/不知道/感到/意识到/开始怀疑" → 这些是内心感受，不是事件
  - "关系升温/气氛微妙/两人之间多了一丝XX" → 这些是状态描述，不是事件
  - 判断标准：如果这个节拍无法被拍成一个具体的**动作镜头**，就不是事件
  - 正确写法：主语+具体动作+可观察的结果。例如：
    - ✗ "苏绾音不确定他是见过还是知道" → ✓ "苏绾音带茶去书房试探，走到门口听见他在里面自语，折返"
    - ✗ "谢沉渊心情复杂" → ✓ "谢沉渊翻出母妃遗物，两枚玉佩并排对比，发现完全一致"
    - ✗ "两人关系更近了一步" → ✓ "苏绾音第一次主动叫他名字，他停下脚步"
- **事件密度自检**：写完【硬】列后，数一下行动节拍数量。如果只有1个，必须补充至少1个"前因行动"或"后果行动"。写作模型没有足够的行动素材时，就会用内心独白凑字数。

**【中】列**：
- 三线中"次要在场"的那条线如何体现
- 感情线具体通过什么行为/对话体现（不允许只写"感情推进"）
- **辅助行动节拍**：如果【硬】列只有2个行动节拍，【中】列必须提供至少1个可选的具体行动/对话场景，确保写作模型总素材量≥3个镜头

**【软】列**：
- **场景发挥提示**（每章 2-3 条，给写作模型提供行动节拍之间的"填充方向"）：
  - 格式：`场景提示：[感官/空间/微动作关键词]`
  - 必须是具体可写的素材，不是抽象描述。写作模型拿到后能直接变成 3-5 句正文
  - 三类提示任选：
    - **感官环境**：这个场景里能闻到/听到/摸到什么？（墨香、廊下风声、纸张质地、茶已凉透）
    - **空间动线**：角色在哪里、怎么移动？（她从廊下走到书房门口停住、他坐在窗边椅子上没有回头）
    - **微动作链**：行动节拍之间角色的小动作？（倒茶→手指在杯沿停→把杯子推过去；笔尖顿了一下→搁笔→抬头）
  - **反面教材**：`场景提示：气氛紧张` ← 这不是提示，写作模型拿到后只会写"空气仿佛凝固"
  - **正面示例**：`场景提示：书房墨香+他在看公文+她倒茶的手势；廊下风声+血玉镯冰凉触感+帖子纸张异常光滑`
- **章末钩子**：必须写，格式为 `**章末钩**：[具体悬置点，标注类型标签]`

#### Step 2b：弧内校验（本弧章节 + 与前弧衔接）

对本弧的所有章节执行以下四项检查，发现问题就地修复：

**① 三线交叉校验**：

- 🔴 单线章：某章仅有一条线有实质动作，其他线完全缺席
- 🔴 日常占位章：【硬】列的事件是"感情日常/散步/用膳"等无事件实质的内容
- 🟡 感情线孤立：感情线推进与当章事件线完全无交叉（只是平行发生）
- 🟡 连续三章缺某线：某条线连续3章以上无任何信号（需回看上一弧末尾章）

**② 钩子设计**：

1. **类型不重复**：最近3章内钩子类型不得完全相同（需回看上一弧末尾章的钩子类型）
2. **强度分布**：本弧最后一章必须是"威胁升级"或"视角切换"型强钩；弧中间章可以是弱钩
3. **感情线+事件线不同时落定**：检查每章，若感情线有阶段收束，事件线的钩子必须更强，反之亦然

**③ 孤岛检查**：

逐章回答以下问题，若答案为"是"则该章为孤岛，必须修改：
- 本章的核心事件在本章内既开始又完全解决（无跨章延续）？
- 本章结尾感情线和事件线都处于"落定"状态（无任何悬置）？
- 本章与前一章、后一章没有任何信息钩连（可以随意调换位置不影响剧情）？

任何一个"是"→修改章节拆分或合并，直到三个问题都回答"否"。

**④ 注水风险检查**：

逐章检查【硬】列，若命中以下任一条则必须补充具体事件：
- 🔴 **行动节拍不足**：【硬】列只有1个行动节拍（写作模型必然会用内心独白凑字数）
  - 修复：补充至少1个前因/后果行动节拍，使总数≥2
- 🔴 **感受型填充**：【硬】列包含感受型动词（不确定/感到/意识到/开始怀疑/心情复杂）而无具体行动
  - 修复：将感受转化为触发该感受的具体事件
- 🔴 **单句事件章**：整章的核心可以用一句对话完成（如"他说了XX"），没有前后行动链
  - 修复：补充"导致这句话的行动"和"这句话之后的行动"，形成 行动→台词→行动 链条
- 🟡 **可合并信号**：相邻两章的【硬】列合在一起仍然只有 2-3 个行动节拍
  - 修复：考虑合并为一章，或从其他章分流事件过来

**注水风险量化标准**：
```
安全：【硬】列有 ≥3 个行动节拍 → 写作模型有足够素材
基本：【硬】列有 2 个行动节拍 → 可能需要【中】列补充
危险：【硬】列只有 1 个行动节拍 → 必然注水，必须修复
```

#### Step 2c：写入文件（追加本弧）

校验通过后，用 Edit 工具在 `大纲/卷{N}细纲.md` 末尾（`<!-- 以下章节细纲` 注释之后，或最后一个弧之后）追加本弧内容：

```markdown

> **弧X（ch{A}-{B}）：[弧名]**

| 章 | 【硬】必须完成 | 【中】主线保持 | 【软】建议/章末钩子 |
|---|---------------|---------------|--------------------|
| {A} | ... | ... | 章末钩：... |
| {A+1} | ... | ... | 章末钩：... |
...

<!-- 弧X 写入完成 -->
```

**写入确认**（每条弧写入后必须输出）：
```
✅ 弧X「{弧名}」已写入（ch{A}-{B}，{已完成数}/{总弧数} 条弧完成）
```

### Step 3：全卷终检 + 三线索追踪表

**所有弧写入完成后**，执行全卷级别的最终检查：

**① 读者视角差检查**：
- 扫描全文，统计【读者视角差】标注数量
- 不足 2 处 → 在合适章末补充，用 Edit 修改对应弧的章节

**② 跨卷大钩检查**：
- 确认卷末最后一章有跨卷级别的强钩子
- 若缺失 → 用 Edit 补充最后一章的章末钩

**③ 感情线独立章检查**：
- 确认全卷无感情线单独占章的情况（都附着在事件弧上）

**④ 追加三线索追踪表**：

用 Edit 工具在文件末尾追加：

```markdown

---

## 三线索追踪

| 章段 | 感情线 | 悬疑线 | 宅斗/对抗线 |
|------|--------|--------|------------|
| {start}-{start+9} | ... | ... | ... |
| {start+10}-{start+19} | ... | ... | ... |
...

---

*表格版：每章3-5行，Context Agent可直接解析*
```

**写入完成后**，输出设计摘要：
```
卷{N}细纲写入完成。
弧结构：[列出每条弧名和章节范围]
视角差设计：[列出安排了哪几章]
需要注意：[任何在设计过程中遇到的约束冲突或不确定点]
```

## 充分性闸门

不得结束流程，除非：

1. ✅ Step 0 的 `preflight` 已成功，`PROJECT_ROOT` 已确定
2. ✅ `大纲/卷{N}细纲.md` 已写入且非空
3. ✅ 每章都有【硬】事件（非日常占位）
4. ✅ 每章【硬】列无感受型节拍（全部为"主语+动作+结果"格式）
5. ✅ 每章【硬】列有 ≥2 个独立行动节拍（事件密度保障）
6. ✅ 每章【软】列有章末钩子且标注了悬置点类型
7. ✅ 全卷无孤岛章（Step 2b 三问均回答"否"）
8. ✅ 全卷无注水风险章（Step 2b 注水检查均通过）
9. ✅ 全卷感情线未单独占章（都附着在事件弧上）
10. ✅ 全卷有 2-3 处【读者视角差】标注
11. ✅ 卷末最后一章有"跨卷大钩"
12. ✅ 三线索追踪表已写入

## 参数说明

调用示例：
- `/write-outline 卷二` → 生成第51-100章细纲（自动从总纲读取范围）
- `/write-outline 卷三 第101-130章` → 只生成指定范围
- `/write-outline 卷一 第29-50章` → 补写或重写卷内指定段落
- `/write-outline 卷二 --resume` → 从断点继续（自动检测已完成弧）
- `/write-outline 卷二 --from 弧D` → 从指定弧开始（跳过前面的弧）

若只传卷号，从总纲自动读取章节范围。
