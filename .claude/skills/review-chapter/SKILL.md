# 执行指令

当用户调用 `/review-chapter` 或要求审查/诊断已有章节时，按以下步骤执行。

**调用格式**：
- `/review-chapter 18` — 审查单章
- `/review-chapter 15-30` — 审查章节范围
- `/review-chapter 18 --focus=感情线,磨损句式` — 聚焦特定维度

禁止仅显示本文档内容而不执行审查。

# Chapter Review（结构化审查工作流）

## 设计目标

1. **输出可执行的修复清单**：每个问题都带精确定位（文件+行号+原文），可直接交给轻量模型执行修复
2. **约束驱动**：所有检查项对照 CLAUDE.md 静态约束 + state.json 动态状态
3. **分级输出**：P0(必须修)/P1(强烈建议)/P2(建议)/P3(可选优化)
4. **跨章检测**：磨损句式、章末同质化、信息传递模式等需要跨章视角的问题

## 输出格式（核心——修复模型直接消费）

最终输出为 Markdown 格式的审查报告，包含两个部分：

### Part A：全局问题（跨章）

```markdown
## 全局问题

### G-001 [P1] 磨损句式：「说不清楚」重复 8 次
- 分布：ch15(×2), ch17(×3), ch18(×3)
- 位置清单：
  | 文件 | 行号 | 原文摘录 | 建议替换 |
  |------|------|---------|---------|
  | 第0015章-xxx.md | 47 | "她说不清楚自己..." | "她也讲不明白..." |
  | ... | ... | ... | ... |

### G-002 [P1] 章末同质化：ch17-19 连续3章「独处+反问」结尾
- 违反规则：CLAUDE.md §五「禁止连续2章以上使用同一结尾模式」
- 位置清单：
  | 文件 | 行号范围 | 当前结尾类型 | 建议改为 |
  |------|---------|------------|---------|
  | 第0017章-xxx.md | 105-115 | 独处+反问 | 行动截断 |
  | ... | ... | ... | ... |
```

### Part B：逐章问题

```markdown
## 第18章 暗流微澜

### C18-001 [P0] 感情线越阶：第 67 行
- 规则：CLAUDE.md §一 动摇期(ch11-25) 禁止「握手腕（非紧急救援）」
- 原文：「他伸手握住她的手腕，将她拉到身后」
- 场景：非紧急救援，日常对话
- 建议：改为不涉及肢体接触的保护方式，如言语警告

### C18-002 [P2] 信息密度偏低：第 80-120 行
- 问题：连续 40 行内心独白，无新信息/无新事件/无新悬念
- 建议：压缩至 15-20 行，或插入一个微事件打断内心戏
```

## 审查维度（完整清单）

### 维度 1：红线检查（P0 级，必须修复）

| 检查项 | 数据源 | 判定标准 |
|--------|-------|---------|
| 感情线越阶 | CLAUDE.md §一 + 当前章号 | 出现超越所在阶段的感情行为 |
| 伏笔泄露 | CLAUDE.md §二 + 当前章号 | 提前揭示/过度暗示受保护伏笔 |
| 道具矛盾 | CLAUDE.md §三 | 道具描述与注册表不一致（颜色/持有者/来源） |
| 预知能力违规 | CLAUDE.md §四 + 当前章号 | 代价缺失/能力超阶/精确度过高 |

### 维度 2：结构性问题（P1 级，强烈建议修复）

| 检查项 | 检测方法 | 判定标准 |
|--------|---------|---------|
| 磨损句式 | Grep 跨章统计 | 同一表达在审查范围内出现 ≥4 次 |
| 章末同质化 | 逐章末尾分类 | 连续 ≥2 章同一结尾模式 |
| OOC（角色失控） | 对照人设核心缺陷 | 苏绾音「心动后无but」/ 谢沉渊情绪外溢 |
| 时间线矛盾 | 跨章时间标记 | 前后矛盾/倒退/大跨度无过渡 |
| 信息传递单一 | 统计信息渠道 | 同一信息传递模式连续 ≥4 次 |

### 维度 3：质量问题（P2 级，建议修复）

| 检查项 | 判定标准 |
|--------|---------|
| 信息密度低 | 连续 ≥30 行无信息增量/情感增量/悬念增量 |
| 场景空转 | 整个场景（---分隔）无情节推进 |
| 对话说明书化 | 连续 ≥5 轮「问-答」无动作/神态/环境插入 |
| 叙事视角混乱 | 同一场景内视角无标记切换 |

### 维度 4：风格优化（P3 级，可选）

| 检查项 | 判定标准 |
|--------|---------|
| 句式单调 | 连续 ≥3 句同一句式结构 |
| 环境描写空泛 | 场景缺乏感官锚点 |
| 排比滥用 | 三句以上排比无收束 |

## 执行流程

### Step 0：解析参数

```
输入：/review-chapter {range} [--focus={维度}]

解析：
- range → start_chapter, end_chapter
- focus → 若指定，只运行指定维度；否则全量运行
```

### Step 1：加载约束上下文

**必读（主 agent 直接加载）**：
1. `${PROJECT_ROOT}/CLAUDE.md` — 全量约束
2. `${PROJECT_ROOT}/.webnovel/state.json` — 动态状态
3. `${PROJECT_ROOT}/大纲/卷一细纲.md`（或当前卷细纲）— 大纲对照

**提取并缓存**：
- 当前章节范围所属的感情线阶段及其禁止行为
- 当前章号下受保护的伏笔清单
- 道具注册表
- 预知能力当前等级

### Step 2：加载章节正文

读取审查范围内所有章节正文。若范围 ≤5 章，直接全文加载。若 >5 章，分批处理（每批 5 章）。

同时加载范围前 2 章的正文（供跨章对比用）。

### Step 3：执行红线检查（维度 1）

**逐章执行**，对每一章：

1. **感情线检查**：
   - 确定本章所属阶段
   - 扫描正文中的肢体接触、称谓、台词、情绪描写
   - 对照阶段禁止清单逐项判定

2. **伏笔检查**：
   - 扫描与受保护伏笔相关的关键词
   - 判断是否存在泄露或过度暗示
   - 特别关注「替嫁知情」相关措辞

3. **道具检查**：
   - 扫描正文中出现的已注册道具
   - 核对颜色、持有者、描述是否与注册表一致

4. **预知能力检查**：
   - 若本章有梦境/闪像场景，检查代价描写
   - 检查能力表现是否超阶

### Step 4：执行结构性检查（维度 2）

**跨章执行**：

1. **磨损句式扫描**：
   - 维护高频表达词典（含已知磨损词：「说不清楚」「目光在她脸上停了一瞬」「温热干燥」「手指在袖中收紧」「目光变得幽深」「老槐树下」等）
   - 对审查范围内所有章节做 Grep 统计
   - 同一表达 ≥4 次即标记

2. **章末同质化检测**：
   - 读取每章最后 10 行
   - 分类为：疑问未答/行动截断/新异常/威胁升级/信息错位/视角切换/独处感悟+反问
   - 连续 ≥2 章同类型即标记
   - 「夜晚 + 独处感悟」需严格判断不被允许

3. **OOC 检测**：
   - 苏绾音：每次心动后是否有「但是」（防备心核心缺陷）
   - 谢沉渊：是否出现超出人设的情绪外溢
   - 周瑾瑜：是否出现超出当前伪装的行为

4. **信息传递模式检测**：
   - 统计信息进入叙事的渠道（谢青来报/春桃闲聊/苏绾音自己发现/偷听/第三方转述等）
   - 同一渠道连续 ≥4 章即标记

### Step 5：执行质量检查（维度 3）

**逐章执行**：

1. 按 `---` 切分场景
2. 逐场景判断信息增量/情感增量/悬念增量
3. 标记流水账段落和空转场景

### Step 6：执行风格检查（维度 4）

**逐章执行**（轻量，可 --focus 跳过）

### Step 7：汇总输出

**输出结构**：

```markdown
# 审查报告：第 {start}-{end} 章

## 概览
- 审查范围：第 {start}-{end} 章
- 发现问题：P0={n}, P1={n}, P2={n}, P3={n}
- 最弱章：第 {n} 章（原因：xxx）
- 最强章：第 {n} 章

## 全局问题
{Part A 内容}

## 逐章问题
{Part B 内容，按章号排列}

## 修复建议优先级
1. [P0] {问题} — {修复方向}
2. [P1] {问题} — {修复方向}
...
```

**报告写入路径**：`${PROJECT_ROOT}/.webnovel/tmp/review_report_ch{start}-ch{end}.md`

## 并行策略

当审查范围 >5 章时，可使用 Agent 并行：
- Agent A：维度 1+3（红线+质量，逐章，前半）
- Agent B：维度 1+3（红线+质量，逐章，后半）
- 主 agent：维度 2（跨章结构性检查，需全量数据）
- 主 agent：维度 4（风格检查，轻量）

汇总时由主 agent 合并所有结果。

## 与修复工作流的衔接

本 skill 的输出格式设计为**可直接被轻量模型消费**：

1. 每个问题都带文件路径 + 行号 + 原文摘录
2. 每个问题都带修复建议（具体到替换内容）
3. 问题按优先级排序，修复模型可从 P0 开始逐项执行
4. 全局问题（磨损句式等）提供完整位置表，修复模型逐行替换即可

**典型修复工作流**：
```
1. Opus 运行 /review-chapter 15-30 → 输出结构化报告
2. 用户确认修复范围（可选择只修 P0+P1）
3. 轻量模型（如 Kimi 2.5）读取报告 + 对应章节，逐项执行修复
4. （可选）Opus 再跑一次 /review-chapter 验证修复结果
```

## 磨损句式词典（持续更新）

> 以下为已确认的高频磨损表达，审查时优先扫描。

| 磨损表达 | 单章上限 | 连续3章上限 | 替代策略 |
|---------|---------|-----------|---------|
| 说不清楚/说不上来 | 1 | 2 | 换为具体的感受描写 |
| 目光在她脸上停了一瞬 | 1 | 2 | 看了她一眼/打量了她/扫过她的脸色 |
| 手指在袖中收紧 | 1 | 2 | 指甲掐进掌心/十指蜷起/掌心攥了一下 |
| 目光变得幽深/深不见底 | 1 | 2 | 眸色暗了几分/目光沉下来/神色冷了 |
| 温热干燥+筑了一道屏障 | 0 | 1 | 每次用不同触感描写 |
| 攥紧XX（段末动作） | 1 | 2 | CLAUDE.md 硬约束 |
| 窗外月色如水/清寒 | 1 | 2 | 换为不同的夜景描写 |
| 独自坐在窗边望着老槐树 | 1 | 2 | 换为不同的独处场景 |
| 心头一紧/心头一震 | 1 | 3 | 换为具体生理反应 |
| 碧绿通透温润无声 | 全书保留1处标准描述 | - | 其余换为不同角度 |

## 禁止事项

- ❌ 审查时修改正文（本 skill 只读不写）
- ❌ 输出模糊建议（如"建议优化感情线"而不给具体行号和原文）
- ❌ 漏检 P0 红线问题
- ❌ 省略行号和原文摘录
