## 三大定律（低自由度 - 必须精确执行）

| 定律 | 规则 | 检查方式 |
|------|------|----------|
| **大纲即法律** | 严格执行大纲，不得擅自发挥 | 审查时对照大纲 |
| **设定即物理** | 实力/招式/物品 ≤ index.db 记录 | 写作前查询确认 |
| **发明需识别** | 新实体由 Data Agent 自动提取 | 章节完成后处理 |

## 新实体处理流程

当前规则：正文不再要求 XML 标签：

1. **写作时**: 直接写纯正文，新角色/地点/物品正常描写
2. **完成后**: Data Agent 自动识别新实体并写入 index.db
3. **不确定实体**: Data Agent 标记为 uncertain，由人工确认

## 章节约束分层

### Hard（必须）

- 本章可读性达标：读者能回答”发生了什么 / 谁在做什么 / 为什么”。
- 本章必须存在清晰推进：问题、目标、代价、关系变化、信息变化至少一项可被识别。
- 若上章有明确承诺（钩子/未闭合问题），本章必须回应（允许部分兑现，不要求一次性结清）。
- 禁止输出占位正文（如 `[待补充]`、`[TODO]`、`...（省略）...`）。
- **感情线与事件线禁止同时闭合**：若本章感情推进达到一个阶段性节点（如确认好感、被护或拥抱），则事件线必须留下至少一个未解疑问；反之亦然。禁止一章内所有线索都”圆满落定”。
- **禁止制造假收尾（最高优先级）**：章末**禁止**做以下任何一件事——
  - 跳跃到夜晚用”月光/反思独白/入睡”来总结白天发生的事
  - 在场景中途人为插入一个与前文场景无关的”升华句”或”意味深长的疑问句”
  - 把正在进行中的场景（对话未结束/行动未完成）强行用内心独白包装成收束
  - 章末的情绪容器（月光/风声/树影）与本章内容没有直接关联，只是作为”氛围装饰”出现
- **禁止日弧结构**：一章不得构成”清晨起床 → 白天事件 → 夜晚反思/入睡”的完整闭合日弧。
  - 禁止以”天亮/次日清晨/醒来/梳妆”作为默认开章方式；若上章结尾时某个场景/对话/事件仍在进行，本章必须**直接承接**该时空，不得跳过一整夜重新开始。
  - **承接优先**：若上章最后一个场景中事件未结束（对话进行中、行动被打断、某人刚到达），本章开头必须是该场景的自然延续。
- **禁止内心独白注水**：角色内心独白中，同一个疑问/信息不得以不同措辞重复超过2次。具体：
  - “还是……还是……还是……”式排比追问最多1组（≤3个分支），禁止连续出现两组
  - “她/他不知道””她/他只知道””没有说话”作为段落收束词，全章合计不超过2次
  - 角色想完一个问题后必须产生行动（哪怕只是一个微动作），禁止连续3段以上纯思绪且无任何动作/对话/感官描写
- **禁止多POV复述同一信息**：当A角色视角已经完整展示了某个信息/事件/对话后，切到B角色视角时：
  - 禁止让B重新叙述A已说过的内容（如A说了”我见过”，B听到后不得再逐字复述”他说了'我见过'”然后反复追问”他见过什么”）
  - B视角只写B的**独特反应和后续行动**，读者已知的信息一笔带过即可（如”她听到了那三个字”）
  - 判断标准：如果删掉B的复述段，读者不会遗漏任何新信息，那这段就是注水

### Soft（建议）

- **章末可以停在场景自然停歇处**，不要求每章都有”悬置疑问句”；一个场景进行到一半是完全合法的章末状态，下章直接继续即可。如果场景自然产生了悬置感，那是加分，但不是必须主动制造的。
- 开头尽早进入冲突/风险/强情绪（建议前 200-400 字，不再固定 120 字）。
- 局面变化建议保持节奏感（参考 800-1400 字一个脉冲，短章至少一次实质变化）。
- 微兑现频率按题材 profile 建议执行，不要求机械等间距。

### Style（可选强化）

- 对话尽量带意图（试探/回避/施压/诱导），减少纯说明句。
- 避免连续大段纯解释；若必须解释，建议切分为“信息 + 行动/反应”。
- 避免“回去休息了”式机械收尾；若使用平缓收尾，需同时保留未闭合期待。

## 爽点与节奏（按题材 profile 调整）

- 爽点密度由题材与章型共同决定：过渡章允许低密度，但不允许“整章无收获”。
- 组合爽点与里程碑爽点采用滚动窗口评估（5章/10-15章），用于预警而非逐章硬判。
- 连续同类型爽点达到 3 章记为风险预警，优先通过类型变体或执行差异化修正。

## Strand 平衡警告

| 情节线 | 警告条件 |
|--------|----------|
| Quest（主线） | 连续 5+ 章 |
| Fire（感情线） | >10 章未出现 |
| Constellation（世界观） | >15 章未出现 |

## 禁止事项

- `[待补充]`、`[TODO]`、`...（省略）...` → 必须完整写出
- 战斗后无善后描述（都市异能题材）
- **流水账** → "谁做了什么、谁又做了什么、再做了什么"的纯事件罗列，每段必须服务于明确的叙事目的（推进/收束/揭示），不得出现无意义的动作序列
